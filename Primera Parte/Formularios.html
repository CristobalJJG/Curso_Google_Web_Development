Formularios

form

Se usa para enviar datos sin representación gráfica.

Atributos de form:

  • action
    • Único atributo requerido.
    • Es necesario mandar una URL.
  • method
    • Forma de envío.
    • GET (por defecto)| POST
      • GET (Formulario de búsqueda)
        • Datos visibles (URL)
        • Limitada
        • Favoritos
      • POST (Formulario de registro)
        • Datos no visibles
        • Sin límite
        • Ficheros
  • accept-charset
    • Codificación de caractéres
      • UTF-8 (Hay que usar este)
      • ISO-8859-1 (Alfabeto latino, poco común)
  • enctype
    • Tipo de Codificación
      • application/x-www-form-urlencoded (por defecto " " = "+")
      • multipart/form-data (subir ficheros mediante formularios)
      • text/plain (convierte los espacios en signos de sumar)
  • input (no tiene etiqueta de cierre)
    • Texto
    • Casillas de verificación
    • Casillas de opción (btón radio)
    • Archivos
    • Texto enmascarado (password)
    • Ocultos
    • Botones
    • Atributos:
      • name:
        • Texto.
        • Nombre del campo.
        • Ej. "input name = "email"/"
      • type (define en qué se convertirá el texto introducido)
        • "text" -> Texto (Por defecto)
          • maxlength = "número"
            • Sirve para acotar a un máximo lo que se vaya a escribir.
          • size = "número"
            • Tamaño del recuadro en número de caracteres.
          • value
            • Valor por defecto de la casilla.
            • Ej. input type = "text" value = "Buscar..."/
        • "checkbox" -> casilla de verificación
          • name = nombre del grupo
            • Todas las casillas que pertenezcan al mismo grupo deberán tener el mismo "name"
            • Ej.
              • input type = "checkbox" name = "temas" /> HTML
              • input type = "checkbox" name = "temas" /> CSS
            • checked = "checked"
              • Si aparece aceptada por defecto.
            • value (importante)
              • Indica lo que debe devolver la casilla que esté activa.
              • Ej. input type = "checkbox" value = "HTML"/
          • "radio" -> casilla de opción (radio button)
            • Serán igual que las checkbox con la diferencia de que solo se puede elegir 1.
            • name = nombre del grupo (importante)
              • Todas las casillas que pertenezcan al mismo grupo deberán tener el mismo "name"
              • Ej.
                • input type = "radio" name = "temas" /> HTML
                • input type = "radio" name = "temas" /> CSS
              • checked = "checked"
                • Si aparece aceptada por defecto.
              • value radio
                • Indica lo que debe devolver la casilla que esté activa.
                • Ej. input type = "radio" value = "HTML"/>HTML
            • "file" -> subir fichero
              • input type = "file" accpecpt = "image/gif"
                • Con accept decimos que tipo de ficheros se pueden agregar.
              • form enctype = "multipart/form-data"
                • enctype tiene que estar bien puesto para que permita recoger un fichero.
            • "password" -> texto enmascarado
                • Mismos atributos que los campos de tipo "text", con la diferencia que al escribir en vez de aparecer letras aparecen "*".
            • "hidden" -> campo oculto
            • "button" -> botón
              • "button" -> Botón sin acción (Para usar este tipo será necesario el uso de JS)
                • value = "texto del botón"
              • "submit" -> botón de enviar
                • Envía el formulario.
                • value = "texto del botón"
              • "image" -> imange como botón de enviar (en desuso)
              • "reset" -> botón de limpiar
                • Reinicia el formulario.
                • value = "texto del botón"
          • disabled
            • Desabilita un campo para que no se pueda ver.
          • readonly
            • Muestra el texto pero no permite editarlo.
      • select
        • Listas desplegables.
        • Listas de selección múltiple.
          • multiple = "multiple"

        Ejemplo de select (listas)

      • textarea
        • cols = "numero"
          • Define el ancho del texArea (20 por defecto)
        • rows = "numero"
          • Define el alto del textArea (2 por defecto)
      • label
        • No muestra nada por pantalla, pero ayuda a la Usabilidad y a la Accesibilidad
      • fieldset
        • Agrupa elementos de un formulario que tienen que ver entre sí.
          • Se le puede dotar del atributo legend, para darle un título al marco

        Atributos para input type:

        url -> Únicamente será valida una URL con su respectivo formato.

        URL

        tel -> Únicamente será valido el patrón (+[0-9]{2} [0-9]{9}) Ej. +34 928928928

             Al usar esta opción en dispositivos móviles se mostrará el teclado numérico en vez de el teclado entero.

           Teléfono:

        email -> Únicamente será válido un correo electrónico con su respectivo formato.

        number -> Únicamente un valor numérico, el cuál se puede acotar mediante "min/max" o se le puede asignar un valor concreto con "value".

        color -> El usuario podrá elegir un color dentro de una paleta de colores.

           Color:

        search -> Añadir un campo de búsqueda de forma correcta.

        range -> El usuario podrá elegir un número dentro de un rango acotado por "min" y "max".

        datetime -> Permite seleccionar la hora, el día, mes y año y la zona horaria.

        datetime-local -> Permite seleccionar lo mismo que en el anterior pero sin la zona horaria.

        date -> Permite seleccionar una fecha con día, mes y año.

           Fecha:

        month -> Permite seleccionar un mes de un año específicos.

        week -> Permite seleccionar una semanade un mes y un año específicos.

        time -> Permite seleccionar la hora con minutos.

           Hora:

        Atributos de contenido:

        autocomplete -> Nos dice si una parte del formulario o este entero debe tener la opción de autocompletado.

            Es posible tener el formulario con autocompletado a on, pero que algún input lo tenga en off

        autofocus -> Nos dice en qué punto debe centrarse primeramente el usuario.

        step -> Indica de cuanto en cuanto puede pasar un valor (step = "2" -> 0, 2, 4, 6, 8...).

        pattern -> Permite realizar un input en base a una expresión regular.

           Prefijo del país:

        placeholder -> Referencia de lo que debería de poner el usuario.

           Nombre:

        required -> Especifica que el input que lo contenga tiene que estar rellenado obligatoriamente antes de enviar el formulario.
